# Governance

## Contexto
Hub de diretrizes corporativas Ávila (procedimentos, arquitetura, compliance, onboarding). Complementa o `ARCHITECTURE.md` com políticas operacionais.

## Objetivo
Garantir alinhamento entre squads, registrar processos críticos e padronizar governança de TI.

## Estrutura Recomendada
- `ARCHITECTURE.md` — visão macro do ecossistema.
- `PROCEDIMENTO_SETUP_GITHUB.md` — onboarding Git corporativo.
- `policies/` — documentos específicos por área.
- `checklists/` — listas de verificação para auditorias.
- `reports/` — atas de comitê, decisões de governança.

## Rotinas Essenciais
- Atualizar este hub a cada reunião do comitê Atlas/Helix.
- Integrar ações com Archivus para registro em `Docs/Relatorios/Auditorias`.
- Referenciar mudanças relevantes no `README.md` raiz.
