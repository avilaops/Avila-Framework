# Architecture Docs

## Contexto
Repositório central de decisões arquiteturais, diagramas, ADRs e mapas de sistemas da ÁvilaOps.

## Objetivo
Garantir que todas as equipes tenham acesso à visão macro (macroserviços, integrações, agentes) e aos registros de decisões estratégicas.

## Estrutura Recomendada
- `adr/` — Architecture Decision Records numerados.
- `diagrams/` — diagramas C4, BPMN, sequência (PlantUML, draw.io).
- `maps/` — mapas de capacidade e dependências.
- `roadmaps/` — planos trimestrais de evolução.

## Rotinas Essenciais
- Logar cada decisão relevante como ADR (template `Shared/templates/adr.md`).
- Manter diagramas versionados (não anexar imagens sem fonte editável).
- Alinhar atualizações com Atlas (governança) e Helix (implementação).
